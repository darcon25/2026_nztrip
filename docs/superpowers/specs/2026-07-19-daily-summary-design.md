# 每日行程摘要 + 地圖連動 設計文件

> 日期：2026-07-19　狀態：待實作

## 目標

在互動地圖下方新增「每日行程摘要」，讓使用者不必點開亮點卡片就能一眼掃過整趟行程。同時把互動地圖與亮點卡片的日期選擇打通，消除目前兩套獨立 Day 按鈕造成的混淆。

## 範圍

兩項變更：

1. **新增每日行程摘要區塊**（手風琴式）
2. **地圖與亮點卡片共用 Day 選擇**

## 一、每日行程摘要

### 元件

新增 `src/components/DailySummary.tsx`，掛在 `App.tsx` 互動地圖區塊下方，自成一個 `<Section id="summary">`。

### 資料來源

沿用 `useData()` 的 `days`（`DayData[]`），不新增 API、不新增 Sheet 分頁。使用欄位：

| 欄位 | 用途 |
|------|------|
| `date` | 收合列左側日期，例：7/25 |
| `day` | 收合列左側 Day 番號 |
| `title` | 收合列標題 |
| `highlights` | 展開後的活動標籤（逗號分隔） |
| `hotel` | 展開後的住宿 |
| `deadline` | 展開後的出發時間（Sheet 的 `alert` 欄） |

不使用 `meals`、`items`、`mapPlace`。

### 版面

手機優先。收合列永遠可見，展開內容點擊後才出現。

```
┌────────────────────────┐
│ 7/22 Day 1 基督城集結 ∨│
│ 7/23 Day 2 基督城整備 ∨│
├────────────────────────┤
│ 7/25 Day 4 探險起點   ∧│
│   ┌────┐┌──────┐        │
│   │露營車││豬五花派│        │
│   └────┘└──────┘        │
│   ⌂ Lakes Edge          │
│   ⏰ 08:00               │
├────────────────────────┤
│ 7/26 Day 5 庫克山     ∨│
└────────────────────────┘
```

- 活動做成 chip 換行排列，不用「·」串長句（手機窄螢幕折行難看）
- 沿用現有 `camp-*` 色票與圓角卡片風格
- 收合列高度 ≥ 44px，符合觸控目標最小尺寸

### 行為

- **一次只展開一天**：點另一天會自動收起前一天
- **預設全部收合**
- 展開/收合狀態純屬本元件內部，不與地圖或亮點卡片共享

### 標題清理

Sheet 的 `title` 欄含 Day 番號且格式不一（`基督城集結 Day 1`、`探險起點：Tekapo (Day 4)`、`蒂阿瑙螢火蟲`）。收合列左側已有 Day 番號，需去除標題中的重複部分，避免出現「Day 4 探險起點：Tekapo (Day 4)」。

處理方式：以 regex 移除結尾的 `Day N`、`(Day N)`、`（Day N）` 及其前方空白，其餘原樣保留。

### 空資料

`highlights` 為空的天（目前 Day 2、3、7、8）展開後顯示灰字「尚未安排」。不自動從 `items` 補、不隱藏該天。

### 明確不做

- 心情標籤（出發的興奮…）— Sheet 無此欄位
- 三餐 — 亮點卡片已有
- 與地圖連動 — 摘要維持獨立

## 二、地圖與亮點卡片共用 Day 選擇

### 現況問題

`AdventureMap.tsx` 與 `Itinerary.tsx` 各自持有 `selectedDay` state 與各自一排 Day 按鈕，互不同步。使用者在地圖選 Day 5 後往下捲，亮點卡片仍停在「全部」。地圖上點地點標記只呼叫 `scrollToHighlights()` 捲動，不篩選日期。

### 做法

把 `selectedDay` 提升到 `App.tsx`，以 props 傳給兩個元件，成為單一資料來源。

- 型別統一為 `number | null`（`null` = 全部）。`Itinerary` 目前用 `'all'` 字串，一併改為 `null`
- 兩排 Day 按鈕都保留（捲到下方時不必往回捲），但顯示同一個選中狀態

### 連動行為

| 操作 | 結果 |
|------|------|
| 在地圖選 Day N | 亮點卡片篩選為 Day N，**畫面自動捲到亮點卡片區** |
| 點地圖上的地點標記 | 篩選為該地點對應的第一天，並捲到亮點卡片區 |
| 在亮點卡片選 Day N | 地圖上 Isa 移到 Day N 的地點，**不捲動**（使用者已在該區） |

自動捲動沿用現有 `scrollToHighlights()` 的 `scrollIntoView({ behavior: 'smooth' })`。

## 已知資料問題（不在程式端處理）

Sheet Days 分頁最後一列 `D11,8/1,測試` 是測試資料，會讓摘要與地圖都多出 Day 11。依「行程以 Google Sheet 為準」的既定決策，程式不做過濾，由使用者自行在 Sheet 刪除該列。

## 驗證方式

1. `npm run lint` 通過
2. 瀏覽器實測：
   - 摘要展開/收合正常，一次只開一天
   - Day 2 展開顯示「尚未安排」
   - Day 4 標題不出現重複的 Day 番號
   - 地圖選 Day 5 → 自動捲到亮點卡片且只剩 Day 5
   - 亮點卡片選 Day 7 → 地圖 Isa 移到米爾福德，畫面不跳動
3. 手機寬度下確認 chip 換行與觸控目標（需用 DevTools 裝置模式，Claude in Chrome 無法真正模擬手機寬度）
