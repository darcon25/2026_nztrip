## 重要決策（別再重議）

## QA 中修過的坑（避免重踩）
- 分帳「記一筆」沒反應 → 根因是 Sheet 預算 id 欄標題空白使 id 變 `undefined`；已改名稱制。
- 互動地圖白畫面 → lucide 的 `Map` 圖示撞名 JS 內建 `Map`；已 alias 成 `MapIcon`。
- **`GOOGLE_MAPS_API_KEY` 在這個 worktree 的 `.env.local`裡是空的，導致全站照片一律 404**：金鑰其實一直都在主目錄 `~/Projects/2026_nztrip/.env.local` 裡是有效的，只是 worktree 分岔出來時沒有帶到這份檔案（`.env.local` 不會被 git 追蹤，每個 worktree 要各自維護）。已從主目錄複製過來修好。**如果又開新 worktree，記得檢查 `.env.local` 是否需要從主目錄複製。**
- 誤判「金鑰過期」：因為瀏覽器對某張圖有本機 HTTP 快取（30 天 `Cache-Control`），舊圖看起來正常顯示，讓人誤以為金鑰還有效；實際上伺服器端當時對所有新請求都回 404。**排查照片問題時，不要只看畫面上「這張看起來沒事」，要直接 curl 後端 API 確認。**
- 同一天多列 Sheet 資料（例如出發前幾天不同家庭分開住宿）如果直接用逗號/頓號拼接會把不同家庭的資料混在一起，要先依「誰」分組再合併，不能全部攤平串成一個字串。
- Google Places API（New）**沒有「封面照片」欄位**，`photos` 陣列的排序不保證把最具代表性的那張排第一——實測 Tasman Holiday Parks 抓到的是遊樂設施照片而非建築外觀，即使該照片解析度已經是該地點最高的（用 `widthPx × heightPx` 排序無法解決，已用 Node 腳本直接呼叫 API 驗證過，Text Search 跟 Place Details 兩個端點回傳順序一致，authorAttributions 也無法區分）。這是 API 本身限制，**沒有可靠的程式解法**，需要特定地點的話只能人工指定圖檔。
- **私人 Airbnb 地址通常在 Google Places 查不到照片**（不是程式問題）——只有公共住宿設施（Holiday Park、飯店等）通常查得到；私人租屋地址請 Google Places 找不到是正常現象。

## 環境/驗證備忘
- 啟動：`cd ~/Projects/2026_nztrip/.claude/worktrees/daily-summary && npm run dev`（port 3000）。lint：`npm run lint`。
- 這個專案的 `tsconfig` 不是 strict 模式，缺少介面欄位（例如 fallback 假資料少了新加的必填欄位）不會擋 `tsc --noEmit`，改介面時仍建議手動確認 fallback 資料有沒有跟著補齊，不能只靠 lint 過關就放心。
- **Claude in Chrome 分頁不在前景（`document.hidden = true`）時，不只是 framer-motion 動畫會凍結，連 `window.scrollTo`／`el.scrollIntoView` 都完全不會生效**（不管 `behavior: 'smooth'` 還是 `'auto'`）——這次驗證導覽捲動修正時卡在這個限制上，沒辦法在這個環境裡對這類行為做真的端對端驗證，只能確認程式邏輯正確、請使用者實機測試。
- Claude in Chrome 的視窗縮放無法真正模擬手機寬度（`innerWidth` 不會跟著變），手機版排版請用瀏覽器 DevTools 裝置模式或真機確認。
- 每次改完務必 `npm run lint` 再 commit。
