## 重要決策（別再重議）

- 家庭身分**用「名稱」**，不用 id 欄（使用者定案；Sheet 預算分頁 id 欄標題空白不可靠）。
- 分帳資料存**網站後端 SQLite**，不寫 Google Sheet（零設定、可稽核）。自煮分工的「煮什麼」也比照辦理。
- 行程**以 Google Sheet 為準**；互動地圖照 Sheet 畫，美術稿排程差異不影響網站。
- 分帳分攤**以家庭為單位、統一 NZD**。
- 景點照片**一律走 Google Maps API 自動抓**，不支援自訂圖片網址。
- 自煮分工「我是哪一家」**存在使用者手機本機（localStorage）**，不設唯一限制——兩支手機可以選同一家，使用者已確認接受這個取捨。

## 環境/驗證備忘

- 啟動：`cd ~/Projects/2026_nztrip && npm run dev`（port 3000）。lint：`npm run lint`（本專案沒有測試框架，lint 就是唯一的自動驗證關卡，等同 `tsc --noEmit`）。
- **Claude in Chrome 的視窗縮放無法真正模擬手機寬度**（`innerWidth` 不會跟著變），要看手機版請用瀏覽器 DevTools 裝置模式；手機 RWD 在程式裡是對的。
- **分頁在背景時地圖標記／捲動動畫會「凍結」不是 bug**：Chrome 對背景分頁會節流動畫。DOM 狀態（例如按鈕有沒有被選中）不受影響、可以照常用程式檢查，但畫面上的動態效果要切到前景分頁才看得出來。
- 每次改完務必 `npm run lint` 再 commit。
- `.env.local` 不會進 git（已在 `.gitignore`）。目前 `main` 這個工作目錄的 `.env.local` 已經設好 `GOOGLE_MAPS_API_KEY` 並實測成功；**worktree 是獨立的檔案系統，不會自動繼承這個檔案**，在別的 worktree 跑要另外設定才會有真實照片（沒設也不會壞，只是變回米色佔位圖）。
