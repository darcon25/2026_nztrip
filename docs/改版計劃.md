# NZtrip 網站改版計劃（手機優先・露營風單頁式）

> 目標設計文件。舊版為深藍夜空風、4 分頁切換的桌面版；新版改為米色露營徽章風、單頁往下捲的手機優先設計。

## 背景

2026 紐西蘭南島家族旅遊網站（React + Vite + Express + SQLite，資料來自 Google Sheet）。團員多用手機瀏覽，舊版在手機上不友善：導覽列擁擠、時間軸浪費空間、每個行程內嵌 Google 地圖 iframe 載入慢又耗流量、簽證頁風格不一致、大量 10px 小字難讀。

改版提案圖：`~/Downloads/APP改版計劃.png`。

## 已定案的設計決策

- 全站米色＋咖啡色露營徽章風；banner 與 logo 圖檔由使用者提供
- 單頁往下捲，頂部 sticky 快速連結列（4 個錨點）：人員班機｜餐廳及煮飯輪值｜住房資訊｜費用分攤&記帳
- **刪除簽證頁**（大家簽證都辦好了）
- 亮點卡片照片：調用 Google 地圖（Google Places API）
- 分帳系統：以**家庭**為單位、幣別**統一 NZD**
- 餐廳輪值、住宿資料：Google Sheet 新增分頁維護
- 互動地圖：使用者以 AI 繪圖生成手繪風南島底圖＋愛蓮那公主角色圖後提供

## 現有可沿用的程式

- `src/components/CommentSection.tsx` ＋ `server.ts` 的 `/api/comments` — 留言板，沿用在亮點卡片上
- `src/components/CuteMap.tsx` — 未使用的互動地圖雛形，「百分比座標定位標記＋SVG 虛線路線」手法是 Phase 3 基礎
- `src/services/googleSheetService.ts` — Google Sheet CSV 讀取（papaparse），擴充新分頁
- `src/DataContext.tsx` — 資料載入與備援機制，擴充新型別
- `server.ts` — Express + better-sqlite3，分帳 API 與照片快取 API 都加這裡

## Phase 1 — 版面與視覺改版（MVP，最優先）

### 1.1 新視覺主題（`src/index.css`）
Tailwind v4 `@theme` 色票：
- 背景米色 `#EFE8D8`（取自 banner）、卡片米白 `#F7F2E7`
- 主文字深咖啡 `#4A2C17`、次要文字淺咖啡 `#8B6F4E`
- 強調色沿用徽章的暖棕／墨綠系
- 移除 `body::before` 的深色夜空背景圖
- 全站元件的 `slate-800/900` 深色 class 換成新色票

### 1.2 單頁式版面（改寫 `src/App.tsx`）
移除 tab 切換，改為由上到下的區塊（各區加 `id` 供錨點）：

```
[Banner 圖（使用者提供，放 public/）]
[sticky 快速連結列：人員班機｜餐廳輪值｜住房資訊｜費用分攤]  ← 捲動時吸頂
[互動地圖區]        ← Phase 3；素材到位前放程式繪製的靜態簡圖
[景點 × 活動 亮點卡片區]
[人員班機區]        ← 沿用 Overview 的抵達班機時間軸＋倒數
[餐廳及煮飯輪值區]   ← 新元件，資料來自 Sheet 新分頁
[住房資訊區]        ← 新元件，資料來自 Sheet 新分頁
[費用分攤&記帳區]    ← Budget 圓餅圖；Phase 2 加分帳
[footer]
```
- **刪除 `src/components/Visa.tsx` 及其引用**，修掉 App.tsx:61 殘留文字
- 錨點跳轉用 `scrollIntoView({ behavior: 'smooth' })`，處理 sticky bar 高度偏移

### 1.3 亮點卡片（`Itinerary.tsx` → 改寫為 `Highlights.tsx`）
取代原本的時間軸＋內嵌地圖 iframe：
- 手機 1 欄、平板 2 欄、桌面 3 欄的照片卡片牆
- 每張卡：景點照片、Day 徽章＋日期、標題、**出發時間**、簡介、**三餐安排**（早／午／晚）、餐廳菜單連結＋熱推餐飲（有才顯示）、「開啟導航」按鈕（跳 Google Maps App）、**收合式留言板**（沿用 CommentSection，點開才載入）
- 頂部保留可橫滑的 Day 1–10 篩選 chips（全部／單日），避免卡片過多讓頁面過長
- **移除所有內嵌地圖 iframe**（手機效能主因），以照片取代

### 1.4 Google 地圖照片（`server.ts` 新增 API）
- 新端點 `GET /api/place-photo?query=地點名`：呼叫 Google Places API (Text Search + Place Photos)，**存進 SQLite 快取**，同地點不重複呼叫（控制費用）
- 需使用者建立 Google Cloud 專案的 **Maps API key**，存入 `.env.local` 的 `GOOGLE_MAPS_API_KEY`（實作時附逐步申請教學）
- 備援：Sheet 加一欄「照片網址」，有填就優先用、API 失敗也用它；都沒有則顯示米色佔位圖

### 1.5 Google Sheet 新分頁與服務擴充（`googleSheetService.ts`）
- 新分頁「**輪值**」：日期、早／午／晚安排、煮飯輪值家庭、餐廳名、菜單連結、熱推餐飲
- 新分頁「**住宿**」：日期區間、住宿名稱、地址（導航用）、入住資訊、房間分配備註
- 現有行程分頁加欄位：出發時間、餐廳菜單連結、熱推餐飲、照片網址（選填）
- `DataContext.tsx` 增加對應型別與備援假資料
- 實作時先給使用者「Sheet 欄位範本」再填資料

### 1.6 手機易讀性通則
- 內文最小 14px（`text-sm`），標籤最小 12px；淘汰 10px 字
- 卡片 padding 手機 `p-4`、桌面 `md:p-6`；按鈕觸控目標 ≥ 44px
- Noto Sans TC 字體與圓角卡片風格保留

## Phase 2 — 分帳系統（費用分攤&記帳區）

### 後端（`server.ts`）
- SQLite 新表 `expenses`：`id, payer_family, amount_nzd, description, split_families(JSON), timestamp`
- API：`POST /api/expenses`、`GET /api/expenses`、`DELETE /api/expenses/:id`
- 家庭名單沿用 Sheet 預算分頁

### 前端（新元件 `Expenses.tsx`，放費用分攤區下半部）
- 「預付分攤」：現有 Budget 圓餅圖照舊
- 「旅途記帳」：
  - 新增表單：誰墊的（家庭下拉）、金額（NZD）、說明、勾選分攤家庭（預設全選、均分）
  - 消費列表（可刪除）
  - **結算 summary**：計算各家庭淨額（墊付－應付），用貪婪法產出最少轉帳筆數的「誰給誰多少」清單，含預付分攤合計
- 幣別統一 NZD

## Phase 3 — 互動地圖（等使用者素材到位）

### 使用者需提供
1. 手繪風南島地圖底圖（直式約 4:5，**不要**畫文字標籤，程式會疊）
2. 愛蓮那公主角色去背 PNG（1–2 個姿勢）

### 實作（新元件 `AdventureMap.tsx`，改造自 `CuteMap.tsx`）
- 底圖上用**百分比座標**放 Day 1–10 地點標記（基督城、蒂卡波、庫克山、皇后鎮、米爾福德、奧瑪魯…），對應實際地理方位
- SVG 虛線路線串起每日路徑
- Day chips 選天數 → 公主角色用 motion 動畫移動到當天地點、該日標記放大顯示重點小卡
- 點標記 → 平滑捲動到下方該日亮點卡片
- 素材未到位前先以程式繪製的簡圖開發測試，素材到了直接換圖

## 驗證方式

1. `npm install && npm run dev`（port 3000）
2. Chrome DevTools 手機模擬（390px）逐區檢查：錨點跳轉、卡片排版、字級、觸控目標
3. `npm run lint`（tsc --noEmit）通過
4. 功能測試：留言新增／刪除、分帳新增消費→結算 summary 人工驗算、Sheet 改資料後重整有更新、無 API key 時照片備援不報錯
5. 用 Claude in Chrome 開頁面截圖確認視覺

## 待使用者提供（不阻塞 Phase 1 開工）

1. Banner 橫幅圖檔＋徽章 logo 圖檔（Phase 1 可先用佔位）

3. Google Sheet 新分頁資料（會先給欄位範本）
4. Phase 3：AI 生成的地圖底圖＋公主角色去背 PNG
